GitHub Setup for All Projects

One repository to rule them all, with proper .gitignore configuration

1

Create Main Repository

Create a new GitHub repository named "code-portfolio" or "my-projects".

# Initialize git in your main folder git init git add . git commit -m "Initial commit: Organized code portfolio" git branch -M main git remote add origin https://github.com/yourusername/code-portfolio.git git push -u origin main
2

Configure .gitignore Properly

Create a comprehensive .gitignore file to avoid uploading unnecessary files:

# OS generated files .DS_Store Thumbs.db # IDE files .vscode/ .idea/ *.swp *.swo # Language-specific ignores # Java *.class *.jar *.war *.ear # Python __pycache__/ *.py[cod] *.so .Python # Node.js node_modules/ npm-debug.log* yarn-debug.log* yarn-error.log* # PHP vendor/ composer.lock .env # XAMPP specific (don't sync the entire xampp folder) xampp/ htdocs/ (except symlinks)
3

Sync Between Devices

On your second device (laptop), clone the repository:

git clone https://github.com/yourusername/code-portfolio.git cd code-portfolio

Always pull before starting work and push when done:

# Before working git pull origin main # After making changes git add . git commit -m "Updated university Java assignments" git push origin main

PHP/XAMPP Solution

Handle PHP projects that need to be in htdocs while keeping them synced

The XAMPP Problem

PHP projects must be inside the htdocs folder to run with XAMPP, but you want to keep all code in one synchronized folder.

1

Solution: Symbolic Links (Symlinks)

Create symlinks from your code-portfolio folder to htdocs:

On Windows:

# Open Command Prompt as Administrator mklink /D "C:\xampp\htdocs\my-php-project" "C:\Users\YourName\code-portfolio\php-projects\my-php-project"

On Mac/Linux:

ln -s ~/code-portfolio/php-projects/my-php-project /Applications/XAMPP/htdocs/my-php-project
2

Alternative: Virtual Hosts

Configure XAMPP to serve PHP projects from your code-portfolio folder:

# Edit C:\xampp\apache\conf\extra\httpd-vhosts.conf <VirtualHost *:80> DocumentRoot "C:\Users\YourName\code-portfolio\php-projects" ServerName local.code-portfolio <Directory "C:\Users\YourName\code-portfolio\php-projects"> Options Indexes FollowSymLinks AllowOverride All Require all granted </Directory> </VirtualHost> # Add to C:\Windows\System32\drivers\etc\hosts 127.0.0.1 local.code-portfolio
3

Multi-Device Sync Strategy

Keep all your code synchronized between PC and laptop

Git Sync Workflow

Use Git as your primary sync tool. All code lives in one repository that you push/pull from both devices.

Cloud Backup

Use GitHub as your cloud backup. Private repositories are free for unlimited projects.

Automation Scripts

Create sync scripts to make the process one-click easy.

1

Create Sync Scripts

Create sync-all.sh (Mac/Linux) or sync-all.bat (Windows) in your root folder:

For Windows (sync-all.bat):

@echo off echo Starting sync process... cd /d "C:\Users\YourName\code-portfolio" echo Pulling latest changes from GitHub... git pull origin main echo Adding all changes... git add . echo Committing changes... set /p commit_msg="Enter commit message: " git commit -m "%commit_msg%" echo Pushing to GitHub... git push origin main echo Sync complete! pause

For Mac/Linux (sync-all.sh):

#!/bin/bash echo "Starting sync process..." cd ~/code-portfolio echo "Pulling latest changes from GitHub..." git pull origin main echo "Adding all changes..." git add . echo "Committing changes..." read -p "Enter commit message: " commit_msg git commit -m "$commit_msg" echo "Pushing to GitHub..." git push origin main echo "Sync complete!"
2

Daily Workflow

Follow this routine when switching between devices:

  1. Before starting work: git pull origin main
  2. Work on your projects
  3. When done: git add .
  4. git commit -m "Description of changes"
  5. git push origin main

3. Work on projects
4. Run sync script before shutting down

Working on Laptop

1. Open terminal in code-portfolio folder
2. Run git pull to sync

4. Run sync script when done

Handling Large Files

For large files (datasets, videos, etc.), use Git LFS or store them outside the repo:
